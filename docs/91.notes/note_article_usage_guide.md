# Note記事作成 使い分けガイド

## 🎯 適切なコマンド選択

### 📊 選択フローチャート

```
記事作成の必要性
    ↓
時間制約は？
    ↓
┌─ 5-10分 → /note_article_quick
├─ 15-20分 → /note_article_standard  
└─ 30分以上 → /note_article (高品質版)
```

### 🚀 コマンド別特徴

| コマンド | 所要時間 | 品質レベル | 適用場面 |
|----------|----------|------------|----------|
| `/note_article_quick` | 5-10分 | 基本 | 情報共有、アイデアメモ |
| `/note_article_standard` | 15-20分 | 標準 | 通常の記事、手順書 |
| `/note_article` | 30-45分 | 高品質 | 公開用、重要文書 |

### 📝 テンプレート直接使用

時間がない場合は、テンプレートを直接使用：

```bash
# 利用可能テンプレート
templates/note_article_template_practical_howto.md       # 実装手順
templates/note_article_template_experience_reflection.md # 体験記
templates/note_article_template_comparison_evaluation.md # 比較評価
templates/note_article_template_ai_agent.md             # AI活用
```

### 🎯 成功のコツ

#### Quick版使用時
- 完璧を求めず、情報共有に集中
- 後で`standard`や`高品質版`にアップグレード可能
- まず書く、後で改善

#### Standard版使用時  
- テンプレートを積極活用
- 品質スコア75点を目標
- バランス重視

#### 高品質版使用時
- 十分な時間確保
- 公開・共有を前提とした品質
- 完全性重視

### 🔄 段階的改善戦略

1. **Quick版**でアイデアを形にする
2. **Standard版**で構造化・詳細化
3. **高品質版**で最終仕上げ・公開準備

この段階的アプローチにより、認知負荷を分散し、継続的な改善が可能になります。

---

**重要**: 使いやすさを優先し、必要に応じて複雑性を追加する段階的アプローチを推奨します。
